# 📡 1.1 네트워크 구성도

- 데이터 센터 네트워크는 **안정적이고 빠른 대용량 서비스 제공**을 목표로 구성
    - 다양한 **이중화 기술**을 사용하여 고가용성 보장
    - 많은 서버와 서비스가 한 네트워크에 연결되므로, **높은 통신량 수용** 필요

- 기존 구성: **3계층 네트워크 구조**
    - Core → Aggregation → Access
    - 트래픽 증가, 복잡성 증가 문제 발생

- 최근 구성: **Spine-Leaf 구조 (2계층 네트워크)**
    - Scale-Out 기반 애플리케이션에 적합
    - **서버 간 통신이 많은 환경**에서 대역폭 균형을 맞추기 위해 등장
    - Leaf 스위치: 서버와 직접 연결
    - Spine 스위치: Leaf 스위치 간 고속 통신 담당

# 🛰️ 1.2 프로토콜

- **프로토콜(Protocol)**: 통신 시 정해진 **규칙(규약)**  
  → 물리적 & 논리적 측면 모두 포함

## 🔹 프로토콜 구분

| 구분 | 설명 |
|------|------|
| **물리적 측면** | 데이터 전송 매체, 신호 방식, 회선 규격 등 (ex. 이더넷) |
| **논리적 측면** | 장치 간 통신을 위한 규칙 (ex. TCP/IP) |

- 제한된 자원으로 통신하므로, **최소한의 데이터**로 **효율적인 프로토콜**을 설계하여 사용

- **TCP/IP 4계층 구조**
    - 애플리케이션 계층: FTP, SSH, TELNET, DNS, SNMP
    - 전송 계층: TCP, UDP
    - 네트워크 계층: IP, ICMP, ARP
    - 링크 및 물리 계층: Ethernet 등

---

# 🌐 1.3 OSI 7계층과 TCP/IP

- **네트워크 구조를 이해**하기 위해 반드시 알아야 하는 개념
- OSI 모델은 통신 과정을 계층화하여 **역할 분담과 표준화**를 지원
- 실제 인터넷에서는 TCP/IP 스택이 사용되지만, OSI 7계층이 개념적으로 더 많이 사용됨

## 🔸 OSI 7계층 vs TCP/IP 구조 비교

| OSI 7계층 | TCP/IP 계층 | 주요 프로토콜 예시 | 데이터 단위 |
|-----------|-------------|---------------------|--------------|
| 7. 응용 계층 (Application) | Application | HTTP, FTP, DNS | Data |
| 6. 표현 계층 (Presentation) | - | SSL, JPEG, MPEG | Data |
| 5. 세션 계층 (Session) | - | NetBIOS, RPC | Data |
| 4. 전송 계층 (Transport) | Transport | TCP, UDP | Segment |
| 3. 네트워크 계층 (Network) | Internet | IP, ICMP, ARP | Packet |
| 2. 데이터 링크 계층 (Data Link) | Link | Ethernet, MAC | Frame |
| 1. 물리 계층 (Physical) | Physical | 전기 신호, 케이블 | Bit |

---

## ✅ 계층별 주요 역할 요약

| 계층 | 역할 |
|------|------|
| 애플리케이션 | 사용자와 가장 가까운 계층. 웹, 이메일 등 |
| 표현 | 데이터 인코딩/디코딩, 암호화 |
| 세션 | 통신 세션 수립/관리/종료 |
| 전송 | 신뢰성 있는 데이터 전송 (TCP), 속도 우선 전송 (UDP) |
| 네트워크 | 데이터가 어디로 갈지 결정 (라우팅) |
| 데이터 링크 | 물리 주소(MAC)를 사용한 전송 |
| 물리 | 실제 전기 신호로 데이터 전송 |

애플리케이션 개발자는 하향식 형식으로 네트워크를 바라봄  즉 L7 -> L1
네트워크 엔지니어는 상향식으로 네트워크를 인식함  L1 -> L7 

# 🌐 1.4 OSI 7계층별 이해하기

---

## 1️⃣ 물리 계층 (1계층)

- 전기적, 기계적, 기능적 인터페이스를 정의
- **전기 신호**를 통한 데이터의 실제 전송을 담당
- 출발지/목적지를 구분하지 않음 (단순 전송)
- 주요 장비: **허브, 리피터, 케이블, 커넥터**

---

## 2️⃣ 데이터 링크 계층 (2계층)

- 물리 계층에서 전송된 신호를 **프레임 단위**로 전송
- **MAC 주소** 기반으로 목적지를 지정
- 동일 네트워크 내에서 **정확한 주소지정 및 충돌 제어**
- 여러 통신을 구분하기 위한 제어 기능 포함
- 주요 장비: **스위치, 브리지**


---

## 3️⃣ 네트워크 계층 (3계층)

- **IP 주소**와 같은 논리 주소를 사용
- 네트워크 간 경로 지정 (라우팅)
- 네트워크 주소와 호스트 주소로 분리 가능
    - 예: `172.31.0.1` → `172.31`(네트워크), `0.1`(호스트)

- 주요 장비: **라우터, L3 스위치**

---

## 4️⃣ 전송 계층 (4계층)

- 데이터의 **신뢰성 있는 전송 보장**
- 패킷 유실, 순서 뒤바뀜, 중복 등을 감지 및 복구
- 시퀀스 번호(순서), ACK 번호(수신 확인) 사용
- **포트 번호**를 통해 여러 애플리케이션 구분

- 프로토콜: **TCP, UDP**

---

## 5️⃣ 세션 계층 (5계층)

- 양 끝단 응용 프로세스 간 **세션 설정, 유지, 종료**를 담당
- 예: 로그인 세션 유지, 다중 접속 제어

---

## 6️⃣ 표현 계층 (6계층)

- 서로 다른 시스템 간 **데이터 표현 형식 변환**
- 암호화, 압축, 인코딩 (ex. MIME, JPEG, SSL)
- 데이터를 표준 형식으로 인코딩하거나 디코딩

---

## 7️⃣ 애플리케이션 계층 (7계층)

- 사용자에게 가장 가까운 계층
- 파일 전송, 이메일, 웹 서비스 등 애플리케이션 간 데이터 교환
- 프로토콜: **HTTP, FTP, SMTP, Telnet, DNS**

---

## ✅ 계층별 주요 프로토콜 및 장비 정리

| 계층 | 주요 역할 | 프로토콜/기술 | 장비 예시      |
|------|-----------|----------------|------------|
| 7. 애플리케이션 | 사용자 서비스 | HTTP, FTP, DNS | -          |
| 6. 표현 | 데이터 표현 변환 | SSL, MIME, JPEG | -          |
| 5. 세션 | 연결 유지 및 종료 | NetBIOS, RPC | -          |
| 4. 전송 | 신뢰성, 흐름 제어 | TCP, UDP | 로드밸런서, 방화벽 |
| 3. 네트워크 | 경로 설정, 라우팅 | IP, ICMP, ARP | 라우터        |
| 2. 데이터 링크 | MAC 주소 기반 전송 | Ethernet, PPP | 스위치        |
| 1. 물리 | 전기적 신호 전송 | UTP, 광케이블 | 허브, 리피터    |

---

📌 **참고**: 실제 네트워크에서는 2~4계층 장비들이 주로 사용되며, 고급 보안 및 로드밸런싱은 7계층(L7) 장비에서 처리하는 경우도 많습니다.
